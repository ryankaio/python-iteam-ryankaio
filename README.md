## Regras de Entrega

1. **Cada desafio tem dois arquivos obrigatórios:**
   - `solucao.py` — seu código funcional
   - `explicacao.md` — sua explicação com suas próprias palavras

2. **A explicação vale tanto quanto o código.** Um código correto sem explicação recebe nota reduzida. Uma explicação clara com pequenos erros técnicos é valorizada.

3. **Commits regulares** são parte da avaliação. Não commite tudo no último dia — o histórico de commits é auditado.

4. **Repositório deve ser público.** Repositórios privados não serão avaliados.
