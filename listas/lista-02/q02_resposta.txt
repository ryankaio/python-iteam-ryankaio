ENUNCIADO:
Para cada cenário, indique a estrutura mais adequada (list/tuple/dict/set) e justifique:
  a) Coordenadas (latitude, longitude) de uma cidade.
  b) CPFs de usuários que já votaram (verificar duplicatas).
  c) Código de produto → preço em um catálogo.
  d) Notas de um aluno ao longo do semestre (pode crescer).
  e) Dias úteis da semana (valor fixo, usado como chave de dict).

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
SUA RESPOSTA:

(
(a) = tuple -> Coordenadas têm quantidade fixa de elementos e normalmente não devem ser alteradas.
(b) = set -> set não permite duplicatas e possui busca muito rápida, ideal para verificar se um CPF já existe.
(c) = dict -> Dicionários armazenam pares chave valor, são uma otima escolha para relacionar código do produto ao preço.
(d) = list -> A quantidade de notas pode aumentar com o tempo. Listas são mutáveis e permitem adicionar novos elementos facilmente.
(e) = tuple -> Os dias úteis são valores fixos e imutáveis.
)
